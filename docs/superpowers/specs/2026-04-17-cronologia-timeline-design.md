# Cronología — Design Spec
**Date:** 2026-04-17
**Route:** `#cronologia`
**Status:** Approved

---

## 1. Purpose

A new page "Azul a través del tiempo" that combines:
- Historical milestones of the city of Azul
- Genealogical curiosities derived from the GEDCOM database
- Discoveries published by Genea Azul (Instagram posts/reels)

Goal: grow audience and provide rich educational content for students, families, and institutions.

---

## 2. Data Structure

Single static file: `data/timeline.json` — array of entry objects sorted by year (then month, then day), oldest first.

```json
[
  {
    "year": 1832,
    "month": 12,
    "day": 16,
    "type": "historia",
    "title": "Fundación de Azul",
    "body": "El coronel Pedro Burgos funda el Fuerte San Serapio Mártir del Arroyo Azul.",
    "source": "Wikipedia",
    "sourceUrl": "https://es.wikipedia.org/wiki/Azul_(Argentina)",
    "storySlug": null
  }
]
```

**Fields:**
- `year` — required integer
- `month` — optional integer (1–12); omit if only year is known
- `day` — optional integer (1–31); omit if only year or year+month is known
- `type` — one of `"historia"`, `"genealogia"`, `"descubrimiento"`
- `title` — short headline
- `body` — 1–3 sentence description
- `source` — display name of source (e.g. "Wikipedia", "Genea Azul IG", "GEDCOM")
- `sourceUrl` — optional URL; omit for GEDCOM-derived entries
- `storySlug` — optional; when set, links to `#historias/<slug>`

**Entry types:**
| Type | Color | Meaning |
|------|-------|---------|
| `historia` | navy (`--ga-primary`) | Historical Azul milestones |
| `genealogia` | gold (`--ga-accent`) | Curiosities from the GEDCOM |
| `descubrimiento` | green (decorative, `#2e7d32`) | Genea Azul discoveries |

## 3. Page Layout and UX

**Structure:** Vertical timeline with a single spine on the left. All entry cards flow to the right. Consistent top-to-bottom reading order.

**Filter tabs** at the top of the section:
```
[Todo]  [Historia de Azul]  [Genealogía]  [Descubrimientos]
```
- Pure CSS class toggling — no re-render
- Active tab styled with `--ga-accent`
- Toggling hides/shows `.ga-tl-entry` elements by `data-type` attribute

**Entry card anatomy:**
- Spine dot colored by type
- Year badge on or near the dot (e.g. `1832`); month/day shown when present (e.g. `16 dic 1832`)
- Card: bold `title`, `body` text (2–3 lines), small `source` link if `sourceUrl` present
- If `storySlug` is set: "→ Leer historia" link navigates to `#historias/<slug>`

**Mobile:** spine stays left, cards go full-width to the right. Filter tabs wrap naturally.

**Performance:** all entries rendered at once from static JSON. Expected ~50–100 entries — no pagination needed.

## 4. Static Data Generation (Backend)

A new `generateTimelineJson()` test method in `GeneaAzulWebResources.java`, following the same pattern as `generateSurnamesJson()` and `generatePersonalitiesJson()`.

Steps:
1. Load hardcoded list of `historia` entries (curated manually from Wikipedia and local sources)
2. Load hardcoded list of `descubrimiento` entries (curated from Genea Azul Instagram posts)
3. Call a new service method to generate `genealogia` entries from the live GEDCOM (~5–8 fixed queries), e.g.:
   - Earliest recorded birth in Azul
   - First recorded immigrant from each major origin country
   - Year the most common surname first appears in Azul
4. Merge all entries, sort by year → month → day
5. Write to `data/timeline.json`

The `genealogia` queries are a fixed set — regenerated only when the GEDCOM is updated, not at runtime.

## 5. Frontend Architecture

**New files:**
- `pages/cronologia.html` — page fragment (lazy-loaded)
- `js/cronologia.js` — rendering and filter tab logic
- `data/timeline.json` — static data (generated by backend test method)

**Modified files:**
- `index.html` — add `<script src="js/cronologia.js">`, add nav item, register route
- `js/router.js` — add `cronologia` route mapping
- `css/main.css` — add `.ga-tl-*` styles

**Module pattern:** `GeneaAzul.cronologia` IIFE with `init()` public method.

**Data loading:** `$.getJSON('data/timeline.json')` called on route entry (no caching needed — single consumer).

**Nav item:** added to `#main-navbar` near "Estadísticas", labeled "Cronología" with `bi bi-clock-history` icon.

---

## 6. Content Plan

**`historia` entries (initial set ~15):** sourced from Wikipedia research already completed — Azul founding, first church, railway arrival, notable institutions, etc.

**`descubrimiento` entries:** user to provide from existing Genea Azul Instagram posts/reels.

**`genealogia` entries:** auto-generated from GEDCOM — ~5–8 entries per generation run.

---

## 7. Out of Scope

- No scraping at runtime — all content is curated into `timeline.json` manually or via backend test method
- No pagination
- No search/text filter within the timeline
- No images per entry (may be added later via an `imageUrl` field)
